User Defined Type

User defined types are aggregate types. The members of the aggregate type are the base types and members of the user-defined type, in definition order. User-defined types are fairly flexible in Wide.

User-defined types are considered complex if they have a non-defaulted or dynamic definition of any of the copy constructor, move constructor, copy assignment operator, move assignment operator, or destructor. If a user-defined type is complex, the aggregate definition of these functions, except the destructor, is not used. Else, these members are implicitly defined as the aggregate versions. The destructor is always defined as the aggregate destructor if there is no definition.

User-defined types are considered dynamic if they have any dynamic member functions or dynamic base types.

  • A user-defined type is copy-constructible if it has a copy constructor defined or if it is non-complex.
  • A user-defined type is trivially copy constructible if it is trivially copy constructible as an aggregate, it is not complex, and it is not dynamic.
  • A user-defined type is move-constructible if it has a move constructor defined or if it is non-complex.
  • A user-defined type is trivially move constructible if it is trivially move constructible as an aggregate and it is not complex, and it is not dynamic.
  • A user-defined type is copy-assignable if it has a copy assignment operator defined or if it is non-complex.
  • A user-defined type is trivially copy assignable if it is trivially copy constructible as an aggregate and it is not complex, and it is not dynamic.
  • A user-defined type is move-assignable if it has a move assignment operator defined or if it is non-complex.
  • A user-defined type is trivially move assignable if it is trivially move constructible as an aggregate and it is not complex, and it is not dynamic.
  • A user-defined type is trivially destructible if it is trivially destructible as an aggregate and it is not complex.
  • A user-defined type may have an implicitly defined default constructor if it is non-complex and there are no members that do not have a visible default constructor or an NSDMI.
  • The context of a user-defined type is the module in which it was defined, or if it was anonymous, the most nested module holding the definition.
  • The size and alignment of a user-defined type is undefined, unless overridden with a size and alignment attribute.
  • A user-defined type is empty if it has no non-empty aggregate members, and it has no dynamic functions, inherited or defined.
  • A user-defined type is constant if it has no non-constant aggregate members, and all trivial properties are true, and it has an implicitly generated or defaulted default constructor.
  • User-defined types may define named members. If there is no named member of a given name, then the base types may be searched. If one base type yields a member, then the result is that; else, if multiple base types yield members which are all overload sets, the result is the union of those overload sets; else, lookup fails.
