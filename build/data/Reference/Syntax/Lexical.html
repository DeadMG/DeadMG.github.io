Wide comes with three complex tokens- identifier, string, and integer. These tokens have variable length and slightly more complex matching rules.

NameRules
IdentifierAn identifier matches any sequence of non-whitespace letters, numbers, or underscores that is not an integer. If the sequence is not prefixed with @, then it must also not be a basic token.
StringA string starts when " is encountered and consumes all input until a non-escaped " is encountered. Any input is permitted between the two quotation marks. As in most C-derived languages, backslashes can be used to escape quotes and produce certain special characters- currently \r, \n, and \t.
IntegerAn integer matches any sequence of numbers.

Wide supports two comment kinds- single-line and multi-line. A single-line comment is introduced by either "//" or "#". The comment token consumes all input until the next newline character is encountered. Multi-line comments begin with the sequence "/*". They last until the sequence "*/" is encountered. Multi-line comment tokens may be nested inside each other.

Any non-whitespace input that does not match one of these token types renders the program ill-formed.
