Aggregate Type

An aggregate type is a classification of types in the Wide type system. Aggregate types have properties based on the properties of their members. Most aggregate types add or change some behaviours.

  • Aggregate types have a public [trivial] copy constructor if there is no member without a visible [trivial] copy constructor.
  • Aggregate types have a public [trivial] move constructor if there is no member without a visible [trivial] move constructor.
  • Aggregate types have a public [trivial] copy assignment operator if there is no member without a visible [trivial] copy assignment operator.
  • Aggregate types have a public [trivial] move assignment operator if there is no member without a visible [trivial] move assignment operator.
  • Aggregate types have a public [trivial] default constructor if there is no member without a visible [trivial] default constructor.
  • Aggregate types are constant if they have no non-constant members.
  • Aggregate types are empty if they have no non-empty members.
  • The destructor of an aggregate type destroys all members in reverse order. It is non-trivial if any member's destructor is non-trivial.

User defined types, arrays, tuples, lambdas, and overload sets are all aggregate types.
